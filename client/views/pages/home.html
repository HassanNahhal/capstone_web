Services

Our solution will allow users to track how much money they spent in a certain category over a specific number of months, as well as start a group which others can be invited to join, and this will allow group admins to see reports for the group.

E-Receipts

Allow people to record transactions without receipts using a single button to let users keep track of small, regularly occurring expenses

Manage Online

Organize data with categories and let the user view reports based on these categories and time

Report & Chart

Generate different styles of graphs and charts to satisfy different users’ preferences for how they like to view their data

Receipt Keeper will be built to work on Android devices and will also include a web app. The mobile application will make it easier for people to manage categories, tags and reports while having the convenience of scanning receipts wherever they are. The web application will provide the list of receipts and show graphs to visualize user’s spendings.

Process

Receipt Keeper is composed of three components; mobile application, web client application, and web server. The mobile application will be built in Android Studio version 2, and will use an open source O.C.R. based on the Tesseract project

  • May 2016

    Plan/ Design

    The web client application will be developed using Angular framework, which is useful for development of single page web app with MVC pattern. Additionally, javascript libraries for uploading image files and visualizing spending will be used

  • June 2016

    Implementation

    The web server application will be implemented on the Loopback framework, and some node modules. LoopBack is an open source framework supported by IBM and will save a lot of time for implementing and managing the API sets. More details about the comparison between BAAS (Backend As A Service)

  • July 2016

    Implementation/ Test

    Additionally, for saving data, Mongo DB will be used. To run the application, DAAS (database as a service) is needed. Among the service providers considered, Mongolab will be used because they provide 500MB of storage space for free which will be enough space during the development phase.

  • August 2016

    Deployment

    Finally, for project management, a GitHub private repository will be used. This will be used not only for source code management, but also managing documents, the project schedule and sharing knowledge within the team - all of this will be fulfilled with GitHub
